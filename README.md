# QA Portfolio – Test Logiciel

Bienvenue dans mon portfolio QA.

Ce repository présente un projet de test logiciel réalisé dans le cadre de ma reconversion vers le domaine de la qualité logicielle.  
L'objectif est de montrer ma capacité à structurer une démarche d'analyse des exigences, de conception de test dans différents contextes et de validation.

---

## Objectifs

Ce projet vise à démontrer ma capacité à :

- Analyser les exigences fonctionnelles
- Concevoir des cas de test structurés
- Identifier les scénarios critiques
- Rédiger une documentation QA 
- Appliquer les bonnes pratiques du test logiciel

---

# Projet 1 : Application SaaS de gestion de rendez-vous médicaux

## Contexte 

Application web permettant aux patients de réserver des rendez-vous avec des professionnels de santé.

## Fonctionnalités testés :

- Authentification et gestion des accès
- Réservation de rendez-vous 
- Gestion des sessions
- Contrôle d’accès et sécurité

---

# Projet 2 : Embedded Project - Landing Gear Safety 

## Contexte 

Simulation d'un module de logiciel embarqué responsable de la gestion du train d'atterissage.  
L'objectif est de démontrer une approche structurée de validation dans un environnement orienté sécurité.

## Fonctionnalités testées 

- Autorisation ou refus du décollage selon l'état du train d'atterissage 
- Condition de rétraction en vol
- Déploiement automatique du train d'atterrissage en phase de descente
- Détection d'incohérences entre capteurs
- Activation d'un état de sécurité en cas d'anomalie

---

# Projet 3 : Financial Flow

## Financial Flows Project

## Contexte

Application interne de gestion de flux financiers (dépôt, retrait, transfert), intégrant des contrôles d’identité et un historique des transactions.

L’objectif est de démontrer une approche structurée :
- Identification et analyse des risques
- Conception des cas de test
- Matrice de traçabilité
- Reporting de test et gestion des anomalies
